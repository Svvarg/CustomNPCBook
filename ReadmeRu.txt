cb = custombooks
<vesrion/status/reload/display/quest-tag/item/jump-to-dim/convert/debug/mapping/check-gui-stat>

cb status состояние маппинга меты книг - существующие диалоги
cb reload - пересоздать маппинг метадаты книг - существующие категории диалогов(Нужно при добавлении новых категорий для новых книг).
cb display - редактирование имени, лора, цвета предмета
cb quest-tag - добавление\удаление к предмету метки делающей его уникальным с невозможностью получить через обычный игровой доступ
cb item - держа предмет в руках Unlocalized Stack и Item names + идишник вещи и её мета\дамаг (Для переводов)
cb jump-to-dim - для дебага перемещение в указанное измерение
cb convert - получить простой текст из большой книги библиокрафта

cx = customextension 
<category/dialog/player-data/player-stat>

cx dialog - набор инструментов упрощающих создание структур диалогов. подкоманды
сx dialog #id satus - данные о диологе по его id
cx dialog #id text - вывести текст диалога в консоль
cx dialog #id gui - быстрое открытие гуи указанного диалога (пока только гуишку категории в которой храниться диалог)
cx dialog #id options - работа с автогенерацией и удобным редактированием "опций-диалога" - вариантов ответов
cx dialog #id script - взаимодействие с внутренними библиотеками скриптов через содержимое диалога. 
    В кастомНпс нет возможности загружать скрипты так же как диалог и соотвествующие им файлы.
    чтобы исключить дублирование и хранение их в nbt существ.

cx player-data - данные CustomNPC об конкретном игроке. 
  подкоманды
cx player-data (player-name) status краткая статистика 
      (Сколько квестов выполнено, сколько взято, кол-во прочитанных диалогов)

cx player-data (player-name) quest - просмотр данных о квестах игрока по указанной категирии и проч
    active | finished [-repeated-only] | is-finished (questId) | catigories

cx player-data (player-name) dialogs - данные о диалогах 
    is-read (dialogId) | readed-in-category (catId) | catigories | total-readed

cx player-stat - получение счётчика статистики для заданного игрока по ключу
    (player-name) (statId|achievementId)
    по умолчанию показывает кол-во использований "книги знаний"

Примеры: 

  Для cx dialog # option:
просмотреть все варианты ответов на диалог номер 16 и краткие данные о самом диалоге
cx dialog 16 status

Просмотреть данные ответа в диалоге номер 8 для варианта ответа номер 4 
(тип заголовок цвет номер диалога(если тип переход на диалог) и команду если задана)
cx dialog 8 options view 4

Редактирование опции ответа на диалог. Доступные команды
cx dialog 9 options edit help             
<title/type/dialog-id/command/color>

Установить цвет первого (0) варианта ответа для диалога с интексом 9
cx dialog 9 options edit 0 color aa00fd


cx dialog 16 option add - Добавление разных типов ответов на диалог

Создать несколько диалогов одной строкой с указанными названиями и поместить их
в соответствующим порядке в варианты ответов для указанного диалога #16. 
В поддиалогах-ответах автоматически создаётся возврат к родителю-узловому диалогу #16. 
Названия создаваемых диалогов разделяются обратным слэшем:
cx dialog 16 option add dialog Вариант # 1 \ Второй варинат \ Третий  

cx dialog # option add [switch|dialog|back|quit|command]
доступные варианты для добавления в ответы на диалог:
  switch - переход на уже существующий другой диалог. 
  dialog - Создать один или несколько диалогов с пустым содержанием но с 
           заданными заготовками и добавить их все как варианты ответа на заданный диалог
  back   - переход назад к указанному номеру диалога         
  quit   - выход из диалога
  command  команда консоли

Удаление одного конкретного варианта ответа в диалоге вместе с самим диалогом
cx dialog (dialogId) option remove (slot)

Удаление сразу всех ответов внутри диалога вместе с самими диалогами
cx dialog (dialogId) option clear
Операции удаления нужно подтверждать (см help для них)

Для большинства команд есть внутреняя справка help:
cx dialog help
cx categories help
